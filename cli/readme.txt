После заливания не забываем npm i

Распределенный реестр хранения данных. Для запуска необходимо настроить путь к сетевому координатору 
(или нескольким узлам) и порт для API (должен быть свободен на ПК и открыт для доступа).

При старте узел синхронизируется с сетью и скачивает все нужные материалы (транзакции, скрипты, контаркты).

Для работы с API используется POST команды на адрес http://host:port/api/:controller (host - адрес узла, port - порт API из настроек).

Мартшруты
--- Получить адреса действующих контрактов.
POST /api/list_contracts
{}

Ответ:
{sucess: true, data: ["Адрес контракта", "Адрес контракта", ...] }

--- Получить список транзакций
POST /api/list_transactions
{}

Ответ:
{sucess: true, data: [{id: "Адрес транзакции", 
    contract: "Адрес контракта", 
    method: "Метод контракта", 
    args: [Аргуметны вызова],
    state: [Состояние транзакции: created, executed],
    result: Результат вызова транзакции (если есть)
    },
    ...
]}


--- Создать контракт
POST /api/create_contract
{"params": {code: "js код контракта: module.exports = class {...}"}}

Ответ:
{sucess: true, id: "Адрес контракта"}


--- Вызвать метод контракта
POST /api/call_contract
{"params": {cid: "Адрес контракта",
    method: "Метод контракта", 
    args: [Аргуметны вызова]}
}

Ответ:
{sucess: true, id: "Адрес транзакции"}


--- Получить результат выполнения транзакции
POST /api/tx_result
{"params": {id: "Адрес транзакции"}}

Ответ:
{sucess: true, data: "Результат выполнения транзакции"}


--- Получить список узлов сети
POST /api/sys/nodes
{}

Ответ:
{sucess: true, data: ["MAC", "MAC", ...]}


Демонстрационная консоль находится по адресу http://host:port/  (http://localhost:3000)
